# 章節開發指引（每章開發必讀）

---

## 這是視頻，不是 PPT

正在做的是**視頻網頁** —— 講者點擊 + 口播 + 錄屏發出去給觀衆看。
判斷每一步做對沒有，標準非常樸素：

- **不像 PPT** —— 觀衆感覺是在看視頻，不是在看翻頁幻燈（頁面中不得包含頁眉頁腳，突出主視覺元素）
- **看起來舒服** —— 配色、字體、節奏都讓人放鬆，不得出現大量的純文字、不得出現字體太小的文字
- **有視覺衝擊** —— 畫面在演事情，不只是文字堆砌，不得一次性全部羅列所有元素，關鍵元素隨進度逐步推進展現

---

## 必須用 CSS / SVG / Canvas / JS 大膽繪製視覺演示

> **這是底線。**
>
> 每一章都至少要有 1~2 處"動起來的圖 / 演示元素"。
> **整章只有純文字 = 驗收不過 = 回去重做。**

視頻感最強的來源 —— 用戶**看見**了被講解的東西在屏幕上演給他看：

- 數字在遞增 / 橫條在生長 / 排名在交換
- 流程節點依次點亮 / 連線自繪
- 對比被一刀切開 / 聚光燈掃過 / 形狀在變形
- 粒子聚攏成形 / 噪聲背景流動 / 字符雨下落
- 模擬終端交互
- 模擬 AI 對話窗口
- 模擬文件目錄樹

**怎麼組合發揮都行 —— 但每章必須用，不允許整章純文字。**

---

## 逐步揭示，禁止一次全展示

整頁內容由**全局 `step` 計數器**驅動 —— 點擊空白處或按 → 鍵推進
一步。設計每一步時心裡要默念：**這一步演什麼，下一步演什麼**。

**最重要的一條**：

> 當口播在說"第一是 X、第二是 Y、第三是 Z"這種**清單 / 列表**時，
> **嚴禁**一個 step 把 X / Y / Z 全部 stagger 上來。

正確做法：

- 一項 = 一個 step
- X 只在它自己的 step 裏獨自亮起
- 講到 Y 時，X 灰化保留作上下文 + Y 亮起
- 講到 Z 時，X / Y 都灰化 + Z 亮起

**判斷標準**：講者會一個一個念出來嗎？會 → 必須逐個揭示。

---

## 內容取捨：抓重點，不要原文搬運

視頻是**音 + 畫**：

- **口播**負責把信息線性講清楚
- **畫面**負責把節拍重點放大、節奏感拉出來

每個 step 屏幕上只掛這個節拍**最值得放大的 1~3 個東西** —— 一個
hero 標語 / 一個數字 / 一組對比 + 必要的視覺演示。

不要試圖把原文每個字都搬上去。那是論文閱讀，不是視頻。

---

## 雙源：節奏跟口播稿，細節回原文章

> **節奏 / 順序 / 節拍切分** 跟 **`script.md` 口播稿** —— **關鍵順序不能亂**。
> **畫面細節 / 數據 / 引用 / 案例** 回 **`article.md` 原文章**抽。

`outline.md` 已經在每章首段抽了「信息池」做參考。但**實現章節時
也必須回去翻 `article.md` 本章對應段落** —— 那裡有比口播稿多得多的
細節（具體數字、引用原話、案例維度、出處時間）。把這些掛到畫面上，
讓**畫面信息密度 > 口播信息密度**。

> **如果你只用了口播稿的內容做章節** —— 屏幕等於把口播打字打了一遍
> —— 那就是 PPT，不是視頻。
>
> 章節實現一定要回原始文章抽細節，不要嫌麻煩。

---

## 字體 / 配色 / 動畫 / 留白 —— 視頻演示基本審美

視頻觀衆離屏幕遠、注意力浮動，所以：

- **字號要大** —— hero 文字至少 80px 起，遠觀也能看清
- **留白要多** —— 舞臺四邊都要讓出大留白，畫面不要塞滿
- **配色要舒服** —— **顏色和字體家族必須用主題 token**（保證換主題不破）；
  字號 / 間距 / 時長這些章節按內容自由發揮（詳見下方「代碼層最小約束」）
- **動畫要舒服 + 炫酷** —— 出現得乾淨利落，停下來不搶戲；炫酷靠
  **設計巧思**（內容驅動的演示動畫），不靠**速度暴力**或**密集閃爍**

---

## 避免 AI 味

AI 生成的網頁有幾種共有的"視覺指紋"，**全部不要**：

- 紫粉 / 藍紫對角漸變背景
- 圓角卡片 + 彩色左邊框裝飾
- 漸變按鈕 + 大圓角藥丸
- emoji 當圖標用
- 假數據 / 假 logo / 假"X 萬用戶"
- 整章 N 步用同一種入場動畫（全場 fade / 全場 blur）
- 每步都掛 ken burns / 光暈呼吸 / 持續閃爍
- 每屏右下角都掛 mono 角標 / 序號

缺的東西**承認缺** —— 用 placeholder 佔位卡（一張寫着"image · 16:9
描述"的卡片，按真實比例留位）。**不要**用 emoji 湊、不要找無關圖湊、
不要編數字。**沒有就承認沒有**，比 fake 強一百倍。

---

## 框架已經搭好的部分（理解就好，不需重寫）

- **16:9 固定舞臺**：內容設計在 1920×1080 上，外層 transform scale
  縮到任何視口，外圍 letterbox 留黑 —— **沒有響應式斷點**
- **舞臺居中 + 大留白**：上下左右四邊都讓出至少 80px 的安全區
- **隱形進度條**：屏幕底部默認完全透明，鼠標懸到底部邊緣才出現，
  支持點擊跳轉章節（錄屏時攝像頭看不到任何 chrome 控件）
- **全局 step 驅動**：點擊舞臺空白處 / 鍵盤 ←/→ 推進；章節是 `step`
  的純函數，沒有定時器、沒有命令式狀態

## 代碼層最小約束

不能踩的紅線，其它怎麼寫都行：

### 必須用 token（換主題不破的底線）

- **顏色**：`--shell` / `--surface` / `--surface-2` / `--surface-3` /
  `--text` / `--text-2` / `--text-mute` / `--text-faint` / `--rule` /
  `--accent` / `--accent-soft` / `--accent-glow` ——
  **禁硬編碼 hex / rgb / 顏色名**
- **字體家族**：`--font-display-cn` / `--font-display-en` / `--font-body`
  / `--font-mono` —— **禁硬編碼字體名**
- **主題性格籤名**通過 primitive class 自動接入，**不要在章節 CSS 裏
  重定義它們**：
  - `.hero-num`（hero 數字風格 —— 主題決定襯線 / 等寬 / 粗黑）
  - `.rule`（分割線 —— 主題決定 1px 實線 / 4px 實線 / 2px 虛線）
  - `.card`（卡片 —— 主題決定圓角 + 陰影性格）
  - `.stage-frame`（舞臺底色 / 圓角 / 陰影 / 裝飾圖案 / vignette
    全自動，章節什麼都不用做）

### 可硬編碼 / 可 token，按內容自由（解鎖章節自由設計）

- **字號**：想要 80px 就寫 80px，想用 `var(--t-h1)` 也行
- **間距 / padding / margin**：按畫面節奏寫具體值
- **動畫時長 / 緩動 / keyframe**：按動畫意圖寫具體值
  （**節奏氣質**參考 `theme.json` 的 `mood` —— 慢主題別寫 200ms 的快動畫）
- **邊框寬度 / 非性格圓角 / 字距**：隨手寫
- **gap / grid 布局尺寸**：按畫面構圖寫

### 其它工程紅線

- 不用 `setTimeout` / `setInterval` 驅動動畫 —— 用 CSS keyframes
- 章節內的可交互元素（按鈕 / 自定義控件）加 `data-no-advance`，
  否則點了會被舞臺誤推進 step
- 章節代碼物理隔離：每章獨立文件夾、獨立 CSS 類前綴，不跨章 import
- **每章必須有 `narrations.ts`**（與 `<Chapter>.tsx` 同目錄）：
  - 數組長度 **=** 章節代碼裏 `if (step === N)` 出現的最大 N + 1
  - 每個元素 = 一個 string，該 step 要播的口播文本（來自 `script.md`
    對應段，**語義一致**——可微調標點 / 斷句以適配 TTS，但不能漏關鍵短語）
  - 完全無音頻的過場 step 用空串 `""`，Auto 模式會按字數估時撐過
  - 這是**音頻合成 + Auto 模式自動推進的唯一真相源**，寫錯或漏寫
    會讓錄屏對不上嘴
- **動畫時長必須 ≤ 該 step 的口播時長**——Auto 模式嚴格按音頻結束推進，
  沒有"等動畫跑完"的兜底。動畫太長 → 三選一：**寫更長口播 / 拆 step
  / 調動畫速度**。詳細機制見 [`AUDIO.md`](AUDIO.md)

## 完工自檢（寫完每章**強制**執行，不可跳過）

> ⚠️ **硬性流程**：章節實現完成後**必須**走完下面的自檢 → 修復 → 匯報
> 三步。**禁止**"實現完成 → 直接匯報給用戶"。
>
> **執行方式**（按能力降級）：
>
> 1. **優先 Agent Teams**：開一個獨立的 reviewer agent，傳入本章代碼路徑
>    + 本文件 Part「完工自檢」清單，讓它**逐項核查 + 出結論**（哪幾條
>    pass / 哪幾條 fail + 證據）。
> 2. **其次 subAgent**：當前 agent 沒有 Teams 能力但能開 subagent，用 subagent
>    走同樣的流程。
> 3. **都沒有**：當前 agent 自己**嚴格逐項**核查，不允許目測一遍就放行。
>
> **拿到自檢結論後**：先按 fail 項**改完代碼**，然後再向用戶匯報"做完
> 了 + 自檢結論 + 改了什麼"。**直接拿原始結論匯報但不修復 = 違規**。

寫完一章 + 在瀏覽器點完一遍後逐項過：

- [ ] **每章至少 1~2 處 CSS / SVG / Canvas / JS 視覺演示** —— 沒有 = 回去補
- [ ] **不同 step 的主導動作不一樣** —— 全章一種動畫 = 回去重做
- [ ] 字號大、留白舒服、配色舒服
- [ ] 清單 / 列表逐個揭示，**1 項 = 1 step**
- [ ] 畫面信息比口播稿多（回了原文章抽細節掛上來）
- [ ] 沒有紫粉漸變 / 圓角彩色邊框 / emoji / 假數據 / 假 logo
- [ ] 缺的素材用 placeholder，不是 fake
- [ ] **顏色和字體家族全部走 token**（無硬編碼 hex / 字體名）；hero 數字
      / 卡片 / 分割線 / 舞臺用 primitive class 接入主題性格 —— 這兩條不
      達標 = 換主題就破
- [ ] 章節交付時**主動告訴用戶**："本章還缺這些素材"
- [ ] 禁止出現小號字體，大量純文字（出現後必須回去改）
- [ ] 禁止出現任何形式的頁眉頁腳，僅展示關鍵內容（出現後必須回去改）
- [ ] **`npx tsc --noEmit` 通過** —— 不通過禁止匯報"做完了"
- [ ] 章節代碼物理隔離：獨立 CSS 類前綴（`.cd-` / `.mg-` / ...），
      未跨章 import，未修改 `chapters.ts` 之外的共享文件
- [ ] **`narrations.ts` 存在**且 `narrations.length` === 章節代碼裏
      `if (step === N)` 用到的最大 N + 1（不一致 = Auto 模式錄屏會錯位）
- [ ] **每條 narration 文本與 `script.md` 對應段落語義一致**（關鍵短語 /
      數字 / 引用全部保留，可爲 TTS 微調標點斷句）—— 錄屏畫外音應當能被
      觀衆聽成同一段稿子
- [ ] **每個 step 的視覺動畫時長 ≤ 口播時長**（口播 `字數 ÷ 4` ≈ 秒數）—— 
      超出會被 Auto 模式當場切斷，動畫演到一半就跳下一步

任一未過 → 回去改。**不要**"先放着以後修"。
